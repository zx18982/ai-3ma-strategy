## 1. 产品定位与目标

### 1.1 一句话定位
一款**纯前端、零部署**的双均线（MA）交叉策略可视化与回测工具，让量化交易工作坊学员用浏览器即可"看图、看信号、看业绩"，并基于真实回测结果获得数据诊断与策略建议。

### 1.2 目标用户
- 主要：量化交易工作坊学员（已具备基础数据/指标认知，正从"看懂指标"走向"写策略"）。
- 次要：对均线策略感兴趣、希望快速验证想法并获取改进建议的小型个人投资者。

### 1.3 核心价值
| 价值点 | 说明 |
|--------|------|
| 直观 | 价格 + 双均线 + 金叉/死叉标记同图呈现，趋势一目了然 |
| 可验证 | 一键回测，输出年化收益 / MDD / 夏普 / 胜率等硬指标 |
| 可对比 | 策略回报 vs 买入持有基准，判断策略是否"真有用" |
| 可解释 | 数据诊断 + 策略分析 + 改进建议，帮助学员理解"为什么赚/为什么亏" |
| 零门槛 | 浏览器双击 `index.html` 即用，无需服务器、无需账号 |
| 可追溯 | 交易记录、每日权益、诊断结论全部可导出，计算过程透明 |

### 1.4 与前期任务的关系
- **Task01 数据引擎**：本工具的数据来源（日线 JSON）。
- **Task02 指标构建**：本工具复用"均线"这一指标概念，并向前延伸到"信号 + 回测"。
- **Task03 本工具**：将指标转化为可决策、可评估、可解释**策略**。

## 2. 用户画像与使用场景

**学员小Quant**（典型一天）：
> "老师讲了双均线，我想拿长江电力在 2023-07 到 2026-07 这段时间试试 5/15 均线到底赚不赚钱。最好能直接看到图上有没有标出买卖点、资金曲线跑赢了基准多少、最大回撤有多大，最后还告诉我这段行情适合用均线策略吗、如果不好该怎么改进。"

**关键场景**：
1. 选一只股票 → 选择回测时间窗口 → 看策略主图与信号分布。
2. 调均线周期（如 11/35）→ 看图形、信号数量与业绩如何变化。
3. 开关交易成本（手续费、滑点）→ 理解成本对收益的真实侵蚀。
4. 点"运行回测" → 看指标卡 + 资金曲线 + 回撤图 + 交易记录。
5. 阅读"数据诊断 + 策略分析" → 理解策略优缺点，并据此调整参数或改用其他策略。

## 7. 交互设计

| 交互 | 行为 |
|------|------|
| 切换股票 | 重新加载数据 → 时间窗口重置为数据全区间 → 重算均线/信号 → 整体刷新 |
| 调节时间窗口 | 起始日期 ≥ 数据最早日，结束日期 ≤ 数据最新日；结束 ≥ 起始；非法时禁用"运行回测" |
| 调节均线周期 | 滑块实时显示数值，释放后重绘主图；长周期必须 > 短周期，否则提示 |
| 切换交易成本 | 开关即时影响回测结果；关闭手续费/滑点显示"理想业绩" |
| 运行回测 | 执行全仓模拟 → 填充指标卡 + 资金曲线 + 回撤图 + 交易表 + 策略分析 |
| 重置 | 参数恢复默认（5/15、全区间、10万、手续费开、滑点关） |
| 主图 hover | tooltip 显示 日期/价格/均线值/持仓状态 |
| 图表缩放 | 滚轮缩放 + 拖拽平移（ECharts dataZoom） |
| 图例点击 | 显隐对应曲线 |
| 交易表 hover | 高亮对应买卖点在主图（可选增强） |

**状态反馈**：数据加载中显示骨架屏/loading；无信号时给出"本区间无金叉/死叉"提示；参数非法给出内联校验错误。

## 8. 数据诊断与策略分析（新增）

本工具不只是"算指标"，还要给出**基于数据特征的解读与可操作建议**。设计为 3 个结论卡：

### 8.1 数据诊断
基于原始数据与参数计算：
- **数据覆盖**：回测区间包含 X 个交易日；数据完整度 100% / 缺失 Y 条。
- **行情特征**：区间整体呈"上涨/下跌/震荡"趋势；年化波动率 Z%。
- **信号分布**：共产生金叉 X 次、死叉 Y 次，平均信号间隔 Z 天。
- **当前状态**：最后一日为"持多/空仓"；最新信号出现在某日期。

### 8.2 策略评价
基于回测指标综合判断：
- **收益能力**：年化收益 X%，跑{赢/输}买入持有 Y%；超额收益来源是抓住主要趋势/还是频繁交易。
- **风险水平**：最大回撤 Z%，夏普比率 W；若夏普 < 1，提示"单位风险收益一般"。
- **交易质量**：胜率 V%，平均持仓 A 天；若交易次数 < 5，提示"统计样本不足"。
- **成本敏感**：手续费/滑点使收益减少 B 个百分点；若成本侵蚀大，提示"需降低换手或选择低费率账户"。

### 8.3 改进建议
根据诊断结果动态生成，典型规则：
- 若策略明显跑输基准且行情呈强趋势：建议"尝试更短周期或改用趋势跟踪更强的参数"。
- 若最大回撤 > 20%：建议"加入止损/止盈或降低仓位"。
- 若信号过于频繁（短周期过小）：建议"放大短周期以减少噪音"。
- 若信号过少（周期过大）：建议"缩短周期或延长回测区间"。
- 若夏普 < 1：建议"该策略在此区间风险收益比一般，可结合其他指标过滤（如 RSI/MACD）"。
- 通用："历史回测不代表未来；震荡市中双均线容易反复打脸，请谨慎使用。"

## 10. 技术架构方案

### 10.1 总体
**纯前端** HTML + CSS + 原生 JavaScript，无需构建工具、无需后端。双击 `index.html` 即可运行（数据通过 `fetch` 读取本地 JSON，建议用本地静态服务器打开以避免 file:// 限制）。

### 10.2 文件结构（与 spec 一致）
```
task03_ma_strategy/
├── web_tool/
│   ├── index.html        # 主页面（结构 + 挂载点）
│   ├── css/style.css     # 样式
│   └── js/
│       ├── app.js        # 主控制器 / 事件绑定
│       ├── strategy.js   # 策略引擎：SMA、信号、回测、指标、诊断
│       ├── charts.js     # ECharts 图表渲染
│       └── analysis.js   # 数据诊断 + 策略评价 + 建议生成
└── data/                 # 已更新的 5 只股票日线 JSON（软链或复制进来）
```

### 10.3 模块职责
- **data loader**：读取 `data/*.json` 与 `stocks.json`，做数据校验（记录数≥30、close 无 NaN、日期升序）。
- **strategy engine（strategy.js）**：
  - `calcSMA(series, n)` → 简单移动平均（前 n-1 项为 null）。
  - `detectSignals(smaS, smaL)` → 金叉/死叉，信号交替约束。
  - `runBacktest(...)` → 全仓买卖模拟，输出每日权益 + 交易记录。
  - `calcMetrics(equity)` → MDD / Sharpe / CumRet / 胜率 等。
- **chart renderer（charts.js）**：封装 ECharts option 构造（净值图、回撤图、价格图）。
- **analysis engine（analysis.js）**：根据数据特征、回测指标、交易成本生成诊断/评价/建议。
- **UI controller（app.js）**：绑定选择/参数/按钮事件，串联"加载→截取区间→计算→绘图→回测→诊断→渲染"。

### 10.4 关键计算逻辑（概要）
- **SMA(n)**：窗口均值；前 n-1 个交易日为 null。
- **信号**：`SMA短[t] > SMA长[t]` 且 `SMA短[t-1] ≤ SMA长[t-1]` ⇒ 金叉(买入)；反向 ⇒ 死叉(卖出)；金叉死叉交替，首信号必为金叉。
- **时间切片**：根据起始/结束日期从完整日线序列中截取子区间，均线与信号在切片后重新计算。
- **回测**：初始空仓；金叉→次日开盘全仓买入；死叉→次日开盘全仓卖出；手续费按成交金额比例扣除，买入取整到 100 股；滑点按成交价比例扣除。
- **指标**：
  - 年化收益 = (1+累计回报)^(252/交易日数) − 1。
  - 最大回撤 = min(权益/历史峰值 − 1)。
  - 夏普 = 日均收益/日收益标准差 × √252（无风险利率取 0）。
  - 胜率 = 盈利循环数 / 总循环数。
  - 成本侵蚀 =（无成本收益 − 实际收益）× 100%。

## 12. 非功能性需求
- **性能**：250 条数据下，均线/信号/回测/诊断计算 < 50ms；图表重绘 < 200ms。
- **兼容性**：Chrome / Edge / Safari 最新版；ECharts 5.5。
- **可维护**：策略计算、图表渲染、诊断生成三者解耦；参数集中在 `spec.yaml` / 配置对象。
- **健壮性**：参数非法、数据缺失、无信号、时间窗口越界等情况均有明确提示，不白屏。

## 14. 验收标准（摘要）
- 5 只股票数据已更新至 2026-07-10，每只 ≥200 条日线。
- 支持选择回测起始/结束日期；日期范围在数据有效范围内。
- 支持开关手续费、滑点，并实时影响回测结果。
- SMA(5)/SMA(15) 正确；金叉/死叉检测正确且交替。
- 主图含收盘价 + 双均线 + 红▲买 / 绿▼卖 标记。
- 回测输出年化收益、MDD（≤0）、Sharpe（年化）、胜率，并与买入持有基准对比。
- 提供数据诊断、策略评价、改进建议三个结论区域。
- 浏览器直接打开即可用；默认展示第一只股票、全区间；交互正常。

## 15. 风险与注意事项
- **本地 fetch 限制**：`file://` 下浏览器可能禁止 `fetch` JSON，需用本地静态服务器（如 `python -m http.server`）；或改为将数据内联/构建时注入。
- **过拟合风险**：学生调参后"看起来很赚"可能只是拟合历史，报告中应提示"历史回测不代表未来"。
- **前复权口径**：所有对比须基于同一前复权口径，避免分红除权造成的假信号。
- **诊断建议局限性**：自动生成的建议基于历史回测统计，不应作为真实投资依据，需在页面显著位置提示。
